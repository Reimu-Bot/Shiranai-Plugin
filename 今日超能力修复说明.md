# 今日超能力插件修复说明

## 修复内容

### 1. 去掉开启开关
- 移除了 `checkEnable(e)` 函数，不再检查 `Config.todaySuperPower.enable` 配置
- 移除了适配器检查逻辑，插件现在可以直接使用

### 2. 修复报错问题
- 修复了 `Config.todaySuperPower.blacklist.id` 可能为空导致的报错
- 添加了安全的配置检查，避免访问未定义的配置项
- 修复了评论长度限制的默认值处理

### 3. 集成审核API替代第三方QQ推送
- 移除了对 `otherBotInfo` 和 `QQBotInfo` 的依赖
- 移除了 `examineReviewInfo` 相关配置
- 集成了 `https://i.elaina.vin/api/审核系统.php` API进行内容审核
- 添加了 Unicode 转中文的功能处理审核结果
- 评论现在会自动通过审核API检查，安全内容直接添加到评论区

## 主要修改文件

### apps/todaySuperPower.js
- 添加了 `unicodeToChinese()` 函数处理Unicode转换
- 添加了 `checkContent()` 函数调用审核API
- 移除了所有 `checkEnable()` 调用
- 简化了评论审核流程，使用API替代人工审核

### config/default_config/todaySuperPower.yaml
- 移除了 `enable: false` 配置
- 移除了 `otherBotInfo`、`QQBotInfo`、`examineReviewInfo` 等配置
- 移除了 `adapter` 适配器配置
- 保留了 `reviewLengthLimit`、`blacklist` 等必要配置

### models/todaySuperPower/index.js
- 简化了 `setScheduleJob()` 方法，移除了第三方QQ推送逻辑
- 添加了安全的配置检查

## 审核API说明

- API地址: `https://i.elaina.vin/api/审核系统.php?text={评论内容}`
- 返回格式: `{"status": "success", "result": {"safe": 1, "reason": "内容安全"}}`
- safe=1: 内容安全，可以通过
- safe=2: 内容不安全，拒绝通过
- reason字段可能包含Unicode编码，会自动转换为中文

## 使用方式

修复后的插件可以直接使用，无需额外配置：
- `#今日超能力` 或 `#你会按下按钮吗` - 查看今日超能力
- `#按下` 或 `#不按` - 选择按钮
- `#评论 内容` - 发表评论（会自动审核）
- `#查看评论` - 查看所有评论
